Media Reference

NavMesh Generation

In this video I show how the navmesh is generated. There are some mandatory variables that need to be set up. In the tool, on the creation parameters, the 'Trace Distance XY' variable allows to change the length of a cell, so The lower the value, the more accurate. You can change the max angle allowed so that AI agents are able to climb ramps in a range from 0 grades to that angle. There is a debug section to visualize the data of the generated navmesh, you can check the current state of the navmesh, what information it has, change the color and other stuff.

NavMesh Rotation

Because the perspective of our world and the orientation from which our character is located can change, we needed our navmesh to be able to be generated in any orientation. In this video I show how two navmeshes are generated in different orientations. To solve the orientation problem what I did was to treat the walls and ceilings as floors, changing the orientation of the volume that generates the navmesh with the orientation of the wall or ceiling where you want to generate it, we will get the navmesh we want.

Smartlinks

A single navmesh limits an area where AI agents are able to walk, so in order to cover a higher area we need to implement smartlinks. A smartlink connects two navmeshes allowing access to the information they contain so that AI agents are able to ask for paths in the connected navmeshes.
